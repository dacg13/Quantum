Issues found — current module
VISUAL — WAVE STATE
Interference pattern too blurry & bloom-heavy
The fringe bands bleed into each other. Real double-slit fringes have sharp bright/dark contrast. The over-blurred glow makes it look like a lens flare, not a physics simulation.
VISUAL — OBSERVER STATE
Decoherence result is too plain — just a smear
When observer is active, the pattern collapses to a featureless cyan blob. It should show two distinct Gaussian humps — the "classical two-slit shadow" — so the contrast with the interference state is dramatic.
TRANSITION
Wave-collapse transition is instant — no drama
Toggling the detector snaps the pattern immediately. The wave function collapse is one of the most dramatic moments in quantum physics — it needs a cinematic wipe/dissolve, not a hard cut.
CONTROLS
Sliders don't visibly affect the pattern
Wavelength and slit separation sliders exist, but moving them doesn't visibly change the fringe spacing in real time. The whole point of the controls is to show that λ/d determines fringe width.
UX — OBSERVER PANEL
"Place Detector" button is not prominent enough
The button blends into the dark card. This is the most important interaction — it should be the most visually dominant element on the right side, not just a small outlined button.
MISSING FEATURE
No intensity graph / cross-section plot
A horizontal intensity profile graph below the main canvas would let users see the exact fringe pattern as a waveform — a standard way physicists actually analyze this experiment.
Suggested improvements
FIX 01 — HIGH
Sharper fringe bands with hard edges
Reduce the bloom radius. Each bright fringe should be a crisp band, dark fringes truly black. Contrast = impact.
FIX 02 — HIGH
Two-hump decoherence pattern
Observer state should render two separate Gaussian peaks aligned with the two slits. Makes the quantum→classical contrast unmissable.
FIX 03 — HIGH
Animated collapse transition
0.8s animation: interference fringes ripple and dissolve into the two humps, with a brief "static noise" frame at mid-transition.
FIX 04 — HIGH
Live intensity profile graph
Add a thin horizontal canvas below the main view showing a real-time waveform cross-section. Updates live when sliders move.
FIX 05 — MED
Wavelength → color mapping
When λ changes, map it to a visible light color (400nm=violet, 700nm=red). The fringe color should shift with wavelength — powerful physics lesson.
FIX 06 — MED
Particle-by-particle build mode
Add a "Shoot particles one by one" toggle. Individual dots land randomly at first, then the fringe pattern emerges statistically — the most mind-blowing version of this experiment.
IDEA 07 — LOW
Fringe order labels (m=0, ±1, ±2)
Small labels on each bright fringe showing the order number. Helps connect the visual to the equation dsinθ = mλ.
IDEA 08 — LOW
Path length difference overlay
On hover over a fringe, show the path difference (Δ = nλ for bright, Δ = n+½λ for dark) as a tooltip. Makes the math tangible.
IDEA 09 — LOW
Sound cue on collapse
A soft "glitch" or decoherence tone when the detector is placed. Subtle audio reinforces the dramatic state change.
